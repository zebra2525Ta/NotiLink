保存を押してもVercelのログ（23:58:46）でずっと [notion] found databases: [] のまま動かないんですね……！

ということは、原因はNotionの権限画面ではなく、Next.jsのAPIコード（/api/memo/route.ts または pages/api/memo.ts）側で完全に確定です。

Notion APIの notion.search は仕様が少し特殊で、フィルターの書き方によっては権限があってもデータベースを1件も返さなくなるバグのような挙動をします。

大誠さん、いま開発しているコード（/api/memo のファイル）を開いて、notion.search を呼んでいる部分を見せてもらえますか？

🔍 おそらくコードがこうなっています
今、コード側でデータベースを探すために、以下のように書いていませんか？

TypeScript
// ❌ 動かない原因になっている可能性が高いコード
const response = await notion.search({
  filter: {
    property: 'object',
    value: 'database' // ← Notionの仕様変更で、これだと空を返すケースがあります
  }
});
🛠️ これを直すための修正コード
もし上記のようになっていたら、以下のようにフィルターを一度完全に外して、全件取得してからプログラム側で絞り込む形に修正すると、一発で未分類などのデータベースを検知できるようになります！

TypeScript
// 🚀 修正版：フィルターを外して検索する
const response = await notion.search({});

// 取得したあとにコード側でデータベースだけをフィルターする
const databases = response.results.filter(item => item.object === 'database');
console.log('[notion] found databases:', databases);
