# NotiLink 仕様書

## 1. プロジェクト概要
自分たちが本当に使いたいアプリをコンセプトに開発。NotionをメインツールとしてAIでサポートするWebアプリ。殴り書きメモ・画像をAIが解析してNotionに自動登録し、1日10件の通知をAIがスケジューリングする。通知はNotionを開くきっかけを作ることを目的とし、ホーム画面ではニュース・株価・天気を毎日表示する。

---

|------|------|

| デプロイ | Vercel |

## 4. ログイン画面

### 概要
NotionアカウントによるOAuth認証のエントリーポイント。メールアドレス登録は不要で、Notionアカウントでのログインのみを提供する。

### UI構成
- アプリ名・ロゴ
- 「Notionでログイン」ボタン1つのみ
- それ以外の認証手段は表示しない

### 仕様詳細
- Notionアカウントを持たないユーザーはログイン不可
- Googleログイン等の他の認証方法は提供しない（Notion前提のアプリのため）
- ログイン成功後は自動的にそのユーザーのNotionワークスペースと連携済みになる
- ログイン時に**ワークスペース全体へのアクセス**を要求する（`scope: "workspace"`）
- セッションはNextAuth.jsが管理

### 認証フロー

| ステップ | 処理内容 |
|----------|----------|
| 1 | ユーザーが「Notionでログイン」ボタンをタップ |
| 2 | NextAuth.jsがNotion OAuthの認証画面にリダイレクト |
| 3 | ユーザーがNotionアカウントでワークスペース全体へのアクセスを許可 |
| 4 | NextAuth.jsがNotionのアクセストークンを自動取得 |
| 5 | Vercel KVにユーザーID＋トークンを保存 |
| 6 | ホーム画面にリダイレクト |

### 事前準備（開発者作業）
- Notion開発者ポータル（notion.so/my-integrations）でOAuthアプリを新規作成
- OAuthを有効化し、リダイレクトURLを設定
  - `https://[アプリURL]/api/auth/callback/notion`
- NextAuth.jsのauthorizationに `scope: "workspace"` を設定してワークスペース全体へのアクセスを要求する

### 必要な環境変数

| 変数名 | 説明 |
|--------|------|
| NOTION_CLIENT_ID | Notionアプリ登録時に取得したClient ID |
| NOTION_CLIENT_SECRET | Notionアプリ登録時に取得したClient Secret |
| NEXTAUTH_SECRET | NextAuth.js用のランダム文字列 |

### エラーケース

| ケース | 対応 |
|--------|------|
| Notionアカウントなし | Notionの登録画面に案内 |
| 認証キャンセル | ログイン画面に戻る |
| トークン保存失敗 | エラーメッセージを表示して再試行を促す |

## 5. ホーム画面

### ショートカット
URLスキームで外部アプリをワンタップ起動。アプリインストール済みの場合はネイティブアプリが起動。未インストールの場合は1秒後にブラウザ版にフォールバック。

| ボタン | URLスキーム | フォールバック |
|--------|------------|---------------|
| Notion | notion:// | https://notion.so |
| Notionカレンダー | notion:// | https://calendar.notion.so |
| SmartNews | smartnews:// | https://smartnews.com |
| SBI証券 | sbisec:// | https://www.sbisec.co.jp |

### 毎日更新される情報

| 表示内容 | 取得元 | 更新タイミング |
|----------|--------|---------------|
| 今日の天気 | OpenWeatherMap API | 毎朝 |
| 今日のニュース | newsdata.io | 毎朝 |
| 注目銘柄3つ | J-Quants API | 毎朝（前日データ） |

## 6. 認証・Notion連携
- Notionアカウントでログインするだけで連携完了
- 他の認証手段は提供しない
- ログインと同時にNotionアクセストークンを自動取得
- トークンはVercel KVにユーザーごとに保存（`userID: token`）
- 開発者がNotionポータルでOAuthアプリを1回登録すれば以降は自動

---

## 7. 入力方法

### テキスト入力
- チャット形式で殴り書き入力
- 会話履歴は毎回クリア

### 画像入力
- スマホカメラまたはカメラロールから画像を選択
- Gemini 2.5 FlashがOCRでテキスト抽出
- 抽出テキストをGroqが構造化してNotionに登録
- シフト表などの読み取りに活用
- 無料枠：1日1500リクエストまで（通常使用では上限に達しない）

---

## 8. AIの処理フロー

| ステップ | 処理内容 |
|----------|----------|
| 1 | テキストまたは画像を入力 |
| 2 | 画像の場合はGemini 2.5 FlashでOCR |
| 3 | Notion APIで各DBのスキーマ（プロパティ名・型）を動的取得 |
| 4 | 取得したスキーマをGroqのプロンプトに含めてJSON形式に構造化・DB振り分け |
| 5 | Notion APIで各DBに登録 |
| 6 | 秘書からの返信をアプリに表示 |

### DB振り分けルール
- Groqはスキーマ情報をもとに入力テキストを解析し、各アイテムを適切なDBに分類する
- プロパティ名・型はスキーマから動的に取得するため、Notionで変更しても自動対応
- 分類できないものはMISC（未分類）DBに登録

### Groqへのプロンプト構成
- 各DBのスキーマ（DB名・プロパティ名・型）を渡す
- 入力テキストをどのDBのどのプロパティに登録するかをJSON形式で返答させる
- JSONのみ返答・余分なテキスト不要を明示する

### 振り分け例
- 入力：「グミ、お菓子、東京」
- 出力：買い物リストに「グミ」「お菓子」、行きたいとこリストに「東京」を登録

## 10. 通知システム

### 方針
- 1日10件をAIが自動スケジューリング
- 通知はNotionを開くきっかけを作ることが目的
- 平日・土日・祝日で構成を分ける
- Notionにデータがある場合はそれを優先
- Notionにデータがない場合はニュース・株価・雑学を通知
- 通知タイプはバナー＋通知センター（OSが自動管理）

### AIが通知に使う情報源

| 情報源 | 優先度 | 内容 |
|--------|--------|------|
| Notionのデータ | 高 | タスク・予定・締め切り |
| 祝日API | 中 | 今日の祝日・記念日情報 |
| 天気API | 中 | 今日・明日の天気 |
| ニュースAPI | 低 | 今日の注目ニュース |
| 株価API | 低 | 前日の注目銘柄・値動き情報 |
| Groq生成 | 低 | 今日の雑学・豆知識 |

### 通知文面のイメージ
- Notionにデータあり：「今日はバイトがあります。14時までに準備を済ませておきましょう」
- 雨の日：「今日は雨です。バイトまで時間があるのでレポートを少し進めておくのはどうですか？」
- 祝日：「今日は〇〇の日です。知っていましたか？せっかくの休日、友達との約束も忘れずに」
- データなし：「昨日値動きが大きかった銘柄です。気になる方はチェックしてみては？」

### 通知枠の構成例（平日）

| 時間帯 | 件数 |
|--------|------|
| 朝8時 | 2件 |
| 午前中（AI決定） | 2件 |
| 昼12時 | 1件 |
| 午後（AI決定） | 1件 |
| 夕方18時 | 2件 |
| 就寝前23時 | 2件 |

### 実装フロー

| ステップ | 処理内容 |
|----------|----------|
| 1 | 毎朝6時にCron Job起動 |
| 2 | Notionデータ・天気・祝日・ニュース・株価をGroqに渡す |
| 3 | AIが当日の通知スケジュールを決定 |
| 4 | Notionに書き戻す |
| 5 | 15分おきのCron Jobが確認して送信 |

## 11. PWA対応
- next-pwaでホーム画面への追加対応
- オフライン時はIndexedDBに一時保存
- オンライン復帰後に自動送信
- HTTPSはVercelが自動対応

---

## 12. セキュリティ
- APIキーはすべてVercel環境変数で管理
- CORS対策としてAPI Routesを経由
- Notionアカウントを持つユーザーのみアクセス可能

---

## 13. 推しポイント
- 通知が毎日違う内容になる「生きている秘書」体験
- 通知がNotionを開くきっかけになる自然な導線
- 殴り書き・写真撮影だけでNotionに登録できる入力の手軽さ
- Notionと戦わず共存する設計
- ホーム画面を開くだけでニュース・株価・天気が確認できる
- 認証・通知・AI・PWA・外部API連携と技術要素が充実
